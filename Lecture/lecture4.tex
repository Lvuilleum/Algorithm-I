\subsection{Divide-and-Conquer Analysis}
The running time $T(n)$ of divide-and-conquer algorithms is typically described by a recurrence:
\[ T(n) = \begin{cases} \Theta(1) & \text{if } n \le c \\ aT(n/b) + D(n) + C(n) & \text{otherwise} \end{cases} \]
Where $D(n)$ is the division time and $C(n)$ the combination time. For **Merge Sort**, this yields $T(n) = 2T(n/2) + \Theta(n)$, resulting in $T(n) = \Theta(n \log n)$.

\subsubsection{Solving Recurrences}
There are three main methods to solve these equations:

\subsubsection{Substitution Method}
We guess the form of the solution and use mathematical induction to find the constants and prove the solution.

\paragraph{Example: $T(n) = T(n/4 + 1) + T(3n/4 - 1) + \Theta(1)$}
\textit{Problem:} Prove $T(n) = O(n)$.
\begin{proof}
We want to show $T(n) \le cn - d$ for some constants $c, d > 0$.
Assume it holds for all values smaller than $n$.
\begin{align*}
T(n) &\le c(n/4 + 1) - d + c(3n/4 - 1) - d + k \quad (\text{where } \Theta(1) \le k) \\
&= cn/4 + c - d + 3cn/4 - c - d + k \\
&= cn - 2d + k
\end{align*}
To satisfy $T(n) \le cn - d$, we need $cn - 2d + k \le cn - d$, which simplifies to $d \ge k$. 
Thus, $T(n) = O(n)$ for a sufficiently large constant $d$.
\end{proof}

Visualizes the cost at each level of subproblems. It is useful for generating a "guess" that can later be verified by the substitution method.

\subsubsection{Master Method}
A "cookbook" for recurrences of the form $T(n) = aT(n/b) + f(n)$. We compare $f(n)$ with $n^{\log_b a}$:
\begin{enumerate}
    \item If $f(n) = O(n^{\log_b a - \epsilon})$, then $T(n) = \Theta(n^{\log_b a})$.
    \item If $f(n) = \Theta(n^{\log_b a})$, then $T(n) = \Theta(n^{\log_b a} \log n)$.
    \item If $f(n) = \Omega(n^{\log_b a + \epsilon})$, then $T(n) = \Theta(f(n))$.
\end{enumerate}

\subsection{Maximum Subarray Problem}
The goal is to find a contiguous subarray with the maximum sum within an array $A$.

\subsubsection{Divide-and-Conquer Approach}
We split the array into $A[low..mid]$ and $A[mid+1..high]$. The maximum subarray is either:
\begin{enumerate}
    \item Entirely in the left half.
    \item Entirely in the right half.
    \item \textbf{Crossing the midpoint}.
\end{enumerate}

The function \texttt{Find-Max-Crossing-Subarray} scans from $mid$ to the left and from $mid+1$ to the right in $\Theta(n)$ time. The recurrence is $T(n) = 2T(n/2) + \Theta(n)$, giving:
