\subsection{The Divide and Conquer Paradigm}

This recursive approach breaks down any problem into three steps:

\begin{enumerate}
  \item \textbf{Divide}: break the problem into smaller subproblems of the same nature.
  \item \textbf{Conquer}: solve each subproblem recursively 
        (base case: direct solution if the problem is small enough).
  \item \textbf{Combine}: merge the partial solutions into a global solution.
\end{enumerate}

\subsubsection{The \textsc{Merge} Procedure}

The key step: merge two already sorted subarrays.
The heads of both lists are compared and the smallest element is placed in the output.
This operation takes \textbf{linear} time $\Theta(n)$.

\subsection{Analysis of Merge Sort}

Let $T(n)$ be the time to sort $n$ elements:
\[
  T(n) =
  \begin{cases}
    \Theta(1)            & \text{if } n = 1, \\
    2\,T(n/2) + \Theta(n) & \text{if } n > 1.
  \end{cases}
\]

\textbf{Recursion tree:} at each level $k$, there are $2^k$ subproblems of size
$n/2^k$, each contributing $\Theta(n/2^k)$. The total cost per level is $\Theta(n)$,
and there are $\log_2 n$ levels, therefore:

\begin{center}
\begin{tikzpicture}[font=\small, >=Stealth]
  \draw (0,0) -- (5,0);
  \foreach \y/\lab/\cost in {
    0/{level $0$: size $n$}/{$cn$},
    -1/{level $1$: size $n/2$ \quad $\times 2$}/{$cn$},
    -2/{level $2$: size $n/4$ \quad $\times 4$}/{$cn$},
    -3/{$\vdots$}/{$\vdots$},
    -4/{level $\log n$: size $1$ \quad $\times n$}/{$cn$}
  }{
    \node[left] at (0, \y*0.75) {\lab};
    \node[right, keygreen] at (5, \y*0.75) {\cost};
  }
  \draw[decorate, decoration={brace, amplitude=5pt}, keyblue]
       (5.8, 0.15) -- (5.8, -3.15) node[midway, right=6pt, keyblue]{$\log n$ levels};
\end{tikzpicture}
\end{center}

\[
  \boxed{T(n) = \Theta(n \log n)}
\]
